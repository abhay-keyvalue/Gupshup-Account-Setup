Gupshup Account Setup Guide

This guide walks you through setting up Gupshup as your WhatsApp Business API provider for the Health WhatsApp Agent backend. Gupshup connects your app to WhatsApp through Meta (Facebook) Business Manager — you cannot skip the Meta steps.

Prerequisites (before you start)

1. Meta (Facebook) Business Manager account

You need a Meta Business Manager account linked to a real business.

  1. Go to business.facebook.com and create or use an existing Business Manager.
  2. Add your business details (legal name, address, website).
  3. Complete Business verification when Meta prompts you (documents such as registration certificate, tax ID, utility bill — depending on country).

Important: Business verification is separate from phone OTP verification. For production messaging and higher limits, Meta usually requires business verification to be approved. Sandbox/testing may work before full verification, but plan for 3–14+ business days for Meta review.

Tips to reduce Meta rejections

  • Legal name and address in Business Manager must match official documents exactly.
  • Use a business website with legal name in the footer; verify your domain in Business Manager if asked.
  • Use a company email on your domain (e.g. support@yourcompany.com), not only a personal Gmail.

If someone else owns the Meta account

  • They must invite you as Admin with full WhatsApp permissions, or
  • During Gupshup onboarding, use “Continue with Facebook” and have the owner complete the popup, or
  • Copy the Direct Facebook Registration link from Gupshup and send it to the Meta account owner.

2. WhatsApp Business phone number

Requirement Details
Dedicated number Must not be active on the regular WhatsApp or WhatsApp Business mobile app on the same number. Migrate or use a new number.
SMS or voice Must receive OTP via text or phone call during setup.
Landline Use Phone call verification if the number cannot receive SMS.
IVR Numbers that only reach an IVR (no human/voicemail pickup) may fail verification.
Country code Use E.164 format without + in API config (e.g. 919876543210, 15551234567).
Display name Shown in chats; must follow WhatsApp display name guidelines.

3. Gupshup account

  1. Sign up at gupshup.io.
  2. Verify your email (OTP during signup).
  3. Accept Terms, Privacy, and WhatsApp commerce policies when prompted.

4. For local development (this repo)

  • ngrok or another HTTPS tunnel for webhooks
  • Backend running on port 3000 (see root README.md)
  • PostgreSQL per backend/.env

Part A — Meta Business Manager & WABA (if not already done)

Do this before or during Gupshup’s “Continue with Facebook” flow.

A1. Create or select a Business Manager

  1. Open Meta Business Settings.
  2. Create a business or select the correct one (top-left business selector).

A2. Create a WhatsApp Business Account (WABA)

  1. In Business Settings, go to Accounts → WhatsApp accounts.
  2. Click Add → create a WhatsApp Business Account.
  3. Enter an account name (internal label).
  4. Choose Your account (your business) or Client’s account if you are an agency (requires client’s Business Manager ID).

A3. Add phone number in Meta (optional pre-step)

Some teams add the number in Meta first; Gupshup embedded signup can also create/link it.

  1. Open the WABA → Phone numbers → Add phone number.
  2. Enter number and display name → complete Meta’s verification OTP.

You can instead complete number + display name entirely inside Gupshup (Part B) — both paths end up linked to the same WABA.

A4. Business verification status

  1. Business Settings → Security Center → Business verification.
  2. Submit documents and track status until Verified (required for full production use).

Part B — Gupshup WhatsApp onboarding (embedded signup)

B1. Start WhatsApp setup in Gupshup

  1. Log in to Gupshup dashboard (or Dashboard → WhatsApp).
  2. Create your first app or open an existing one.
  3. Click Go Live (or Complete FB Registration if you started before).
  4. Choose phone number type (new number vs migration — follow prompts).
  5. Review data storage region and pricing → Proceed.
  6. Enter business details, accept policies, verify email OTP → Confirm.

B2. Connect Meta — “Continue with Facebook”

  1. Click Continue with Facebook (or share the registration URL with your Meta admin).
  2. Allow browser pop-ups — Meta opens in a popup; do not close it.
  3. Log in to Facebook or Continue as [name].
  4. In Meta’s flow: Get started → fill business information → Next.
  5. Create or select a WhatsApp Business Account (WABA) and business profile.
  6. Enter display name and category → Next.
  7. Complete Meta phone OTP (SMS or call) → Next.
  8. Review permissions → Continue → Finish.
  9. Back in Gupshup: Confirm WABA ID and phone number → Go to dashboard.

Your app should show as live (or pending display-name review).

B3. Add your WhatsApp phone number (Gupshup UI)

This matches the Gupshup screen “Add your WhatsApp phone number” (Step 3 in the wizard):

Choose how you want to be identified

Option When to use
Use a display name only Limited identity mode; users may only see display name in chat. Read Gupshup’s tooltip before choosing.
Add a new number (recommended for API) Full number registration; verification required.

If you select Add a new number:

  1. Country code — e.g. US +1, IN +91.
  2. Phone number — digits without country code in the field (country code is separate).
  3. WhatsApp Business display name — must match your brand; follows Meta guidelines (e.g. Your Clinic Name, not generic “Support”).
  4. Verify your number
    • Text message — default for mobile numbers.
    • Phone call — use for landlines that cannot receive SMS.
  5. Enter the OTP Gupshup/Meta sends → complete wizard → Next.

You’ll receive a code to verify this number. Keep the phone nearby during setup.

B4. Display name and number approval

  • New display names often show Pending review in Meta WhatsApp Manager until approved.
  • Messaging may be limited until the name and number are approved.
  • Check status in Meta: WhatsApp Manager → your WABA → Phone numbers.

Part C — Get API credentials for this project

After the app is live, collect these four values for backend/.env:

Env variable Gupshup source Example
GUPSHUP_API_KEY App Settings (gear) → API key; or My Wallet → profile → API key 61b3021c...
GUPSHUP_SOURCE Your registered WhatsApp number (digits only, with country code) 15557106411
GUPSHUP_APP_NAME App name (src.name in API) — exact match from dashboard HealthAgentApp
GUPSHUP_TEMPLATE_WELCOME Template UUID from approved template (not display name) c6aecef6-bcb0-...

Where to find them

  1. WhatsApp dashboard → select your app.
  2. Settings → note App name, App ID (WABA-related id), API key.
  3. Phone number / Overview → note the connected sender number → use as GUPSHUP_SOURCE (no +).

Sandbox (testing only)

  • Sandbox may use Gupshup’s proxy number (e.g. 917834811114) and opt-in flow.
  • Use the Sandbox tab in the dashboard for the test sender number and opt-in instructions.
  • Production must use your own verified business number.

Part D — Configure inbound webhook (callback URL)

Gupshup sends user messages and events to your callback URL.

D1. Public URL

Environment Callback URL
Local https://<your-ngrok-subdomain>.ngrok-free.app/api/webhook
Production https://<your-backend-host>/api/webhook

Start ngrok:

ngrok http 3000

Set PUBLIC_URL in backend/.env to the same HTTPS base (no trailing slash).

D2. Set callback in Gupshup

  1. Dashboard → your app → Overview (or Integration → Webhooks).
  2. Open Webhooks tab.
  3. Enter callback URL → Set / Save.
  4. Enable events you need:
    • Inbound messages (required for this agent)
    • Delivery / read (optional, for status tracking)

Note: Gupshup recommends responding to webhooks with HTTP 200 and an empty body. A JSON or text body in the response can cause unintended auto-replies to users. When you migrate the backend to Gupshup, use res.status(200).end().

D3. Test webhook

  1. Start backend: cd backend && npm install && node server.js
  2. Send a WhatsApp message to your business number from a personal phone.
  3. Check backend logs for inbound payload.
  4. Confirm the message appears in Gupshup’s message/logs UI.

Part E — Message templates (first contact / outside 24h)

WhatsApp requires approved templates to message users who have not messaged you in the last 24 hours.

  1. In Gupshup dashboard, open Templates for your app.
  2. Create a template (e.g. utility greeting) → submit for WhatsApp approval.
  3. After approval, copy the template ID (UUID), not only the friendly name.
  4. Set GUPSHUP_TEMPLATE_WELCOME in .env or pass templateId to POST /api/users/welcome after backend migration.

See Gupshup template messages.

Part F — Backend environment variables

Add to backend/.env (replace placeholders after Gupshup + Meta setup):

# Gupshup Configuration
GUPSHUP_API_KEY=your-gupshup-api-key
GUPSHUP_SOURCE=15557106411
GUPSHUP_APP_NAME=YourAppName
GUPSHUP_TEMPLATE_WELCOME=template-uuid-from-dashboard

# Admin WhatsApp for escalation alerts (digits only, country code, no +)
ADMIN_WHATSAPP_NUMBER=919876543210

# Public HTTPS base URL (required for webhooks + outbound media URLs)
PUBLIC_URL=https://your-ngrok-or-production-url

# Grok API (unchanged)
GROK_API_KEY=your-grok-api-key

# Database (unchanged)
DATABASE_URL=postgresql://postgres:postgres@localhost:5433/healthagent

PORT=3000

Remove WATI variables after migration is complete:

  • WATI_API_BASE_URL
  • WATI_API_TOKEN
  • WATI_TENANT_ID
  • WATI_CHANNEL_PHONE

Update render.yaml production env vars to use GUPSHUP_* instead of WATI_*.

Part G — Admin escalation notifications

The backend can WhatsApp-alert an admin on safety escalations.

  1. Set ADMIN_WHATSAPP_NUMBER (digits only, with country code).
  2. Admin must message your business number at least once to open a 24-hour session (or you send them an approved template first).
  3. Trigger a test: user sends a message containing “emergency” → admin should receive the alert.

Verification checklist

Meta / Facebook

  • Meta Business Manager created
  • Business verification submitted (and approved for production)
  • WABA created or linked
  • Business phone number verified (SMS or call)
  • Display name approved or pending (tracked in WhatsApp Manager)

Gupshup

  • Gupshup account created and email verified
  • WhatsApp app Go Live / FB registration completed
  • API key, app name, and source number documented
  • Callback URL set to https://.../api/webhook
  • At least one template approved (for welcome / cold outreach)
  • Wallet/billing configured if required by your plan

This project

  • backend/.env updated with GUPSHUP_* variables
  • PUBLIC_URL matches ngrok or production host
  • Backend running; test inbound message received
  • Test outbound reply from agent or admin dashboard
  • ADMIN_WHATSAPP_NUMBER tested for escalation (optional)

Important operational notes

24-hour session window

  • Within 24h after the user’s last message: send free-form session messages (text, media).
  • Outside 24h: use template messages only.

Migrating from another BSP (e.g. WATI)

  • Disable two-factor authentication on the number at the old provider before migrating via Gupshup embedded signup.
  • Plan a cutover window: update webhook URL, env vars, and disable the old provider webhook to avoid duplicate processing.

Outbound media (videos from admin dashboard)

  • Gupshup sends media using a public HTTPS URL in the API payload (not multipart upload like WATI).
  • Ensure PUBLIC_URL serves /uploads/... over HTTPS so Gupshup can fetch the file.

India data localization (if applicable)

  • During onboarding, Gupshup may prompt for data storage region (e.g. India).
  • Choose the region that matches your compliance requirements; this is set at signup, not in application code.

Troubleshooting

“Continue with Facebook” popup blocked

  • Enable pop-ups for gupshup.io and facebook.com.
  • Try another browser or disable strict blockers.

Cannot create WABA — business not verified

  • Complete Meta Business verification in Security Center.
  • Wait for approval before expecting full API access.

Phone verification fails

  • Number still registered on WhatsApp mobile app → delete account from app or use a different number.
  • Try Phone call instead of SMS for landlines.
  • Ensure country code matches the physical number.

Webhook not receiving messages

  • Callback URL must be HTTPS and publicly reachable.
  • Confirm URL in Gupshup matches running backend (/api/webhook).
  • Restart ngrok; update URL if the subdomain changed.
  • Return 200 with empty body from webhook handler.

API errors (401 / invalid apikey)

  • Regenerate or copy API key from app Settings.
  • Use the app-level key tied to the live WhatsApp app, not an unrelated wallet key (unless your setup uses wallet-level keys).

Messages send but user does not receive

  • Check template vs session rules (24h window).
  • Check display name / number status in WhatsApp Manager.
  • Review Gupshup delivery webhooks / message logs for failed events.

Display name rejected

  • Use legal or well-known brand name; avoid generic words (“Support”, “Test”).
  • Align with website and Meta business name.
  • Resubmit in WhatsApp Manager after fixing the profile.
